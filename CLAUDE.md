# LIMITLESS Digital Twin

API-only microservice for the LIMITLESS Longevity OS platform. Stores and serves a user's health digital twin: health profile, biomarkers, wearable data, activity log, and AI context snapshots.

## Stack

- **Runtime:** Fastify 5 + TypeScript (ESM)
- **ORM:** Drizzle ORM + postgres.js driver
- **Database:** PostgreSQL + TimescaleDB extension (wearable_data is a hypertable)
- **Auth:** JWT cookie (`payload-token`) from PATHS, scoped to `.limitless-longevity.health`
- **Package manager:** pnpm (do NOT use npm, yarn, or bun)

## Commands

```bash
pnpm dev          # Start dev server with tsx watch
pnpm build        # TypeScript compile to dist/
pnpm start        # Run compiled output
pnpm lint         # ESLint
pnpm test         # Vitest
pnpm db:generate  # Generate Drizzle migration
pnpm db:migrate   # Run Drizzle migrations
pnpm db:push      # Push schema directly (dev only)
```

## Architecture

- **No frontend** — API only, consumed by PATHS, HUB, and other LIMITLESS services
- **Entry point:** `src/index.ts` — Fastify server with plugin/route registration
- **Auth plugin:** `src/plugins/auth.ts` — reads `payload-token` cookie, verifies JWT
- **Database:** `src/db/schema.ts` (Drizzle schema), `src/db/client.ts` (singleton)
- **Routes:** `src/routes/` — each file registers its own route prefix
- **Migrations:** `drizzle/` — generated by drizzle-kit

## Hard Constraints

- NEVER commit `.env` files or echo secrets in CI (repo may be public)
- NEVER use `db:push` against production — use `db:generate` + `db:migrate`
- ALWAYS validate request bodies with Zod before database operations
- ALWAYS check `request.user` for auth before accessing user data
- ALWAYS use `.js` extensions in relative imports (ESM requirement)
- Pass `{ ssl: { rejectUnauthorized: false } }` when connecting to Render/Neon databases in CI
